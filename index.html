Hosting

There are a lot of different options for site hosting Here are some of the more common services and some comparisons within each of them

Shared Hosting

Shared Hosting refers to a partition on a master web server. This means that you own a little slice of some company's server, but you do not have complete permissions and control over it.
Many shared hosting services vary in features. Some of them may only allow basic file transfer through an FTP client, so you have to copy and paste from your local computer to the remote server.
Other services might allow SSH access (pay extra?), or they might have Git pre-installed. This all depends on the individual service, and is rarely listed anywhere - you really have to start paying and using the server to figure out what you can and cannot do.
The biggest problem with shared hosting is that you are never allowed sudo access. This makes it very difficult to install services or frameworks through the ssh terminal, so if you're looking to do that kind of stuff, shared hosting isn't for you.
One thing shared hosting is really good for is installing basic CMS like Wordpress. Shared hosting may be restrictive, but they also provide pre-installed tools like CPanel, PHPMyAdmin, which are all services you can use to manage your database through their web application. This is good for people who don't want to fiddle with the command line.

Shared Hosting Examples

  • GoDaddy: $7.99 $3.99/month (sale right now!)
  • Lunarpages: $9/month + extra for SSH and other features
  • Media Temple: $20/month, includes email forwarding etc.
  • Site5: $5 - $12/month
So here are some examples of shared hosting services and their pricing plans. Again, it's pretty difficult to differentiate between them, but some important things are email aliases, SSH, price and web user interface.

Dedicated Hosting

Dedicated hosting is the level up from shared hosting. Dedicated hosting gives you complete freedom over your own server. This is suitable for people who are building richer web apps, with custom frameworks, databases, etc.
It requires a lot of fiddling around with the command line and permissions, so is definitely not for people who aren't familiar with that. However, if you are, I highly recommend it, because it's about the same price and offers that much more flexibility.
Here's an example to illustrate the difference between Shared and Dedicated hosting:

Installing Wordpress on Shared Hosting

  1. Download and upload Wordpress onto server (FTP)
  2. Create a database and MySQL User through CPanel/PHPMyAdmin
  3. Run Wordpress install by accessing site URL

Installing Wordpress on Dedicated Hosting

Note: this is for a fresh server

  1. Configure Root Login
  2. Configure SSH Access
  3. Install LAMP (Linux, Apache, MySQL, PHP)
  4. Set up phpMyAdmin
  5. Download Wordpress onto server (wget)
  6. Create a database and MySQL User
  7. Run Wordpress install by accessing site URL

Dedicated Hosting examples

  • Digital Ocean: $5/month or $0.007/hour
  • Heroku: "scales with you", $0.05/dyno-hour
  • AWS: 1 year free!
  • Linode: $20/month
    • So if the previous example didn't scare you away, dedicated hosting is for you!
    Again, each different service has its own features and quirks, it's up to you to research and find the one that suits you best.
    I personally use Digital Ocean, and have used Heroku at work - both are pretty sweet to work with, after you get past the hassle of the initial setup.

Look and Feel

And finally, we're on to look and feel.
The most important thing about look and feel is that a lot of people, including myself, always have grand visions of their dream site and end up falling short.
I always envision a blog, a portfolio, fancy online resume, slideshows and carousels and stff, custom animations.
But the truth is a lot of this stuff is already built and available out there, and there's no point in inventing the wheel.
I often find that it is a lot easier to design with content in mind, instead of laying out an amazing theme skeleton with nothing to fill it in.
So that was just general advice regarding designing a site.
